# TG_bot_glucose_level
Telegram-бот для ведения дневника диабетика
Этот бот помогает людям с диабетом отслеживать уровень сахара, дозы инсулина и потреблённые хлебные единицы. Записи разделяются на дневные и ночные. Бот формирует отчёты за выбранный период в формате Excel и строит график динамики сахара.

Функциональность
Внесение дневной записи – пользователь последовательно вводит:

уровень сахара (число с плавающей точкой);

дозу инсулина (число);

количество хлебных единиц (целое число).

Внесение ночной записи – аналогично дневной, но хлебные единицы автоматически устанавливаются в 0.

Отчёт за 7 дней – формируется Excel-файл со всеми записями за последние 7 дней и график изменения уровня сахара.

Отчёт за 45 дней – то же за 45 дней. Если за указанный период данных нет, бот присылает все имеющиеся записи либо сообщает об их отсутствии.

Данные хранятся в базе MySQL с автоматическим созданием таблицы при первом запуске.

При ошибках соединения с БД выполняется автоматическое переподключение.

Используемые технологии
Python 3.7+

pyTelegramBotAPI – взаимодействие с Telegram Bot API

mysql-connector-python – работа с MySQL

pandas – обработка данных и создание Excel-отчётов

matplotlib – построение графиков

openpyxl – запись Excel-файлов

python-dotenv – загрузка переменных окружения из файла .env

Установка и запуск
1. Клонирование репозитория
bash
git clone https://github.com/androdjinn/TG_bot_glucose_level
cd diabetes-bot
2. Установка зависимостей
Желательно использовать виртуальное окружение:

bash
python -m venv venv
source venv/bin/activate  # для Linux/Mac
venv\Scripts\activate     # для Windows
Установите необходимые пакеты:

bash
pip install -r requirements.txt
Если файла requirements.txt нет, создайте его со следующим содержимым:

text
pyTelegramBotAPI
mysql-connector-python
pandas
matplotlib
openpyxl
python-dotenv
3. Настройка базы данных MySQL
Создайте базу данных (например, diabetes_bot) и пользователя с правами на неё. Запомните имя базы, пользователя и пароль.

4. Создание файла .env
В корне проекта создайте файл .env и укажите в нём следующие переменные:

text
BOT_TOKEN=ваш_токен_бота_от_BotFather
DB_USER=пользователь_mysql
DB_PASSWORD=пароль_mysql
DB_HOST=хост (по умолчанию localhost)
DB_NAME=имя_базы_данных
Пример:

text
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
DB_USER=diabot
DB_PASSWORD=securepass
DB_HOST=localhost
DB_NAME=diabetes_bot
5. Запуск бота
bash
python bot.py
Бот начнёт опрашивать серверы Telegram и будет готов принимать сообщения.

Использование
Отправьте боту команду /start. Появится меню с кнопками:

Внести дневной – начало ввода дневной записи.

Внести ночной – начало ввода ночной записи.

Отчет за 7 дней – получение отчёта за последние 7 дней.

Отчет на 45 дней – получение отчёта за последние 45 дней.

При выборе дневного или ночного ввода бот последовательно запрашивает показания.

Для дневной записи: сахар → инсулин → хлебные единицы.

Для ночной записи: сахар → инсулин (хлебные единицы = 0 автоматически).

После успешного сохранения бот возвращает меню.

Отчёты приходят двумя сообщениями: файл Excel (.xlsx) и изображение графика.

Структура базы данных
Таблица records создаётся автоматически при первом запуске. Содержит поля:

user_id (BIGINT) – идентификатор пользователя Telegram.

sugar_level (FLOAT) – уровень сахара.

insulin_dose (FLOAT) – доза инсулина.

bread_units (INT) – количество хлебных единиц.

timestamp (DATETIME) – дата и время записи.

nocturnal (TINYINT) – признак ночной записи (1 – ночь, 0 – день).

Логирование
Ошибки и важные события записываются в консоль (уровень ERROR). Формат лога:
время - уровень - сообщение.

Возможные проблемы и их решение
Ошибка подключения к БД – проверьте параметры в .env, доступность сервера MySQL, создана ли база.

Бот не отвечает – убедитесь, что токен верный, бот запущен и не заблокирован.

График не отправляется – возможно, не хватает памяти для создания изображения; проверьте логи.

Неверный формат ввода – бот запросит повторить ввод, если вместо числа введён текст.
